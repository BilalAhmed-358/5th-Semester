Question:

                     -: Design an O(n log n) algorithm :-

Given the input array :

x0 x1 x2 x3 x4 x5....x2n

use merge sort to it in O(n log n) time to produce a sorted list

xa0 xa1 ...xa2n

where the indicate the permutation that you have performed on the intial list to obtain the second list.

I claim then that the pairing that produces the minimum maximum sum over all pairingnis the following pairing:

(xai, xa2n-i) for i = 0, 1, 2 , 3, 4, 5 .....That is, you group the highest lavued number with the lowest valued number available.

Proof-

Proceed by  induction, for the case 2n = 2 this is obvious.

Without loss of generality consider that the input is a list of sortd numbers (since if it is not then sort them)

x0  x1 x2  x3  x4  x5 .....x2n.

Condider the pairing of  x2n with any number, then clearly the minmum sum of this pairing is achieved with (x2n, x0).

Now consider the pairing of x2n-1, either (x2n, x0),(x2n-1, x1) is the pairings that produce the minumin max sum, or (x2n, x1),(x2n-1, x0) is, or both are.

In the latter case our choice dosen't matter. In the seconds to last case, this is impossible (think about it).

In the general case if we proceed inductively by this process, when we are looking for a pair for x2n-k, xk is the lowest unused value that we can pair up with, however; suppose insread we pair up xk with some other x2n-j  for j<k, to try to get a lower minum sum.

This is impossible as, x2n-j + xk >= x2n-k + xk, so at most we can only acieve the same minimum sum.

This  means that choosing (x2n-k, xk) gives us the minimum pairings.
