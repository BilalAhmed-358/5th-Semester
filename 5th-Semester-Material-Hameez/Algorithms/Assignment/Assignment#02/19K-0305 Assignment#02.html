def CountSort(array):
    MaxValue = max(array)
    
    #Declare new array of max size
    newArray = [0] * (MaxValue+1)

    for index in range(0,len(array)):
        newArray[array[index]]=newArray[array[index]]+1 
    
    i=j=0
    while(i<MaxValue+1):
        if(newArray[i]>0):
            array[j]=i
            newArray[i] -= 1
            j += 1
        else:
            i += 1

import numpy as np
import pandas as pd


def findSquare(num):
    if num == 0:
        return 0
    if num < 0:
        num = -num
    
    #find floor value by bitwise right operator (n/2)
    half = num>>1
    
    if(num%2==0):
        return (findSquare(half) << 2)
    else:
        return (findSquare(half) << 2) + (half << 2) + 1

Pros and Cons of JUMP SEARCH¶

Pros:¶

  • It seaches fewer item in the list than the linear search by jumping fixed number of steps.
  • The optimal jump searches requires root(n) time to search the element.
  • It requires less comparisions than the linear search

Cons:¶

  • It only works for sorted input array.
  • It takes more time than binary search in cases as log(n) < root(n)
  • If the jump steps are larger than the number of comparisions will be very high, so it performs good when jump steps are suitable as compared to the size of the array.
  • performs like linear search if array is of small size.

Pros and Cons of BINARY SEARCH¶

Pros:¶

  • It searches faster than both the linear search and the jump search.
  • It takes way less number of comparisions when array is large enough.
  • Time complexity of binary search is (logn) which is smaller than both the linear and jump search

Cons:¶

  • It's more complicated to code binary search than the linear search.
  • It doesn't performs well in smaller arrays as the mid element will not be large enough to divide and search.
  • Only works on sorted list.

Interpolation Search:¶

interpolation search is just an improvement for the binary search. It is used in the situations where the values in a sorted array are distributed uniformly, that is, the distance between the values are uniform. Interploation search may go to different array indexes depending upon the value it gets from the formula: pos = lo + [ (x-arr[lo])*(hi-lo) / (arr[hi]-arr[Lo]) ]

it search for the position and determines the position/location of the search element. If the key is in starting indexes, the position is most likely to return starting index position and it will search starting indexes. It first checks the position that it has found, if that's the position. It returns that index, if it's less than the position, it checks out new position on left side and right side respectively dependending upon if the value is larger or smaller.

Exponential Search:¶

exponential search or doubling search is a searching algorithm for searching infinite/ unbounded lists. The idea behind this search is to find the range where the search element might lie and to perform binary search in that particular range so it performs the search pretty fast. It only works on sorted array. In the first stage, if the list is in ascending order, the code looks for very initial exponent i and the value of 2^j is greater than the search key for which this 2^j becomes the upper bound for the binary search with the previous power of 2^j -1 which becomes the lower bound for the binary search. In the second stage, the binary search is applied on the range.
